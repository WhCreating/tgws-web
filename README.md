# Tg-Ws Web Interface v1.0.1
Локальный веб-интерфейс для tgws прокси

**!ВНИМАНИЕ**: Все права принаджелат оригинальному автору: ![Flowseal tg-ws-proxy](https://github.com/Flowseal/tg-ws-proxy)

# Запуск

## Linux
1. Установите докер командой:  
**Ubuntu**: `sudo apt install docker docker-compose`  
**Arch**: `sudo pacman -S docker docker-compose`  
**Alpine**: `sudo apk add docker docker-compose`  
**Fedora**: `sudo dnf install docker docker-compose`

2. Просто запустите докер композ командой: `docker compose up -d`

## Windows
1. Установка докер: https://www.docker.com/products/docker-desktop/
2. Запуск осуществляется точно так же, как и на линукс: `docker compose up -d`

Вам возможно потребуется войти ещё в докер

# Использование

- Веб-интерфейс откроется на порту: 8000
- Открыть интерфей можно по ссылке: 127.0.0.1:8000, если он запущен на том же хосте, если вы захотите открыть его с другого устройства в локальной сети, то вместо ip: 127.0.0.1 введите ip устройства на котором запущен веб-интерфейс. Получить ip устройства можно командой: `ipconfig` на windows или `ip a` на linux, ищите ip, начинающий на 192.168
- У web-interface есть два эндпоинта: / и /admin, на эндпоинте / можно получить прокси, а на /admin можно перезагрузить прокси, но он требует пароль.
- **Как работает /admin:** вы вводите пароль на этом эндпоинте и нажимаете перезагрузить, если вы впервые запустили сервер, то пароль установиться новый. Если забыли пароль, то вы всегда его можете изменить на хосте в файле app/your_password.json, и как вы уже догадались, пароль не хэшируется, так как я не вижу в этом смысла.
